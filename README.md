# Solar Nations 2 Wiki

Auto-generated data encyclopedia for [Solar Nations 2](https://store.steampowered.com/app/2275440/Solar_Nations_2/). Built with [Astro](https://astro.build/) and deployed to GitHub Pages.

## Quick start

```bash
npm install
npm run build:local   # extract + normalize from game install, then build
npm run preview       # preview at http://localhost:4321
```

For day-to-day development after data is generated:

```bash
npm run dev
```

## Refreshing game data

The wiki reads normalized JSON from `data/curated/`. To regenerate from your local game install:

```bash
npm run data          # runs extract + normalize
```

Set `SN2_GAME_PATH` if your install is not in the default CrossOver Steam location:

```bash
export SN2_GAME_PATH="/path/to/twilightModernity"
npm run data
```

Default install path:

```
~/Library/Application Support/CrossOver/Bottles/Steam/drive_c/Program Files (x86)/Steam/steamapps/common/Solar Nations 2/Windows/twilightModernity/
```

See [docs/extraction.md](docs/extraction.md) for full base-game extraction via FModel.

## Project structure

```
data/
  curated/          # Normalized wiki-ready JSON (committed for CI)
  raw/              # Raw FModel exports (gitignored)
docs/
  extraction.md     # How to extract full game data
public/
  assets/           # Copied flags and icons (regenerated by normalize)
scripts/
  extract.ts        # Scan UE5 paks, write extraction manifest
  normalize.ts      # Parse defines → curated JSON + copy assets
src/
  components/       # StatTable, ModifierList, PrerequisiteChain, etc.
  layouts/          # BaseLayout
  lib/              # Types, categories, data loaders
  pages/            # Index, category lists, entry detail pages
```

| `tutorialMod/modifiers.json` | 174 modifier definitions |
| `tutorialMod/missioncomponents.json` | 40 mission components |
| `Saved/Flags/` | ~502 country flags |
| `Saved/Icons/` | Game icons |
| `data/raw/Defines/` | Full base-game export (via FModel) |

## Deploy to GitHub Pages

1. Create a GitHub repo (e.g. `Solar-Nations-Wiki`)
2. Commit and push to `main`
3. Enable GitHub Pages: Settings → Pages → Source: **GitHub Actions**
4. Push triggers `.github/workflows/deploy.yml` automatically

Update `site` and `base` in `astro.config.mjs` if your GitHub username or repo name differs.

## Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start dev server |
| `npm run build` | Build static site (uses committed curated data) |
| `npm run build:local` | Regenerate data from game + build |
| `npm run extract` | Scan UE5 asset index |
| `npm run normalize` | Normalize game data to `data/curated/` |
| `npm run data` | extract + normalize |
